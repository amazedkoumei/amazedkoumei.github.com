原文

GitHub loves third party developers. We want to help make your project awesome, and often that includes showing off GitHub integration with an image. In order to keep everyone happy (and avoid involving lawyers), we've created this repository and filled it with artwork that you may use in your own applications.

For help or to add media not included here, please email support@github.com and we'll talk.

 

THESE THINGS ARE AWESOME, PLEASE DO THEM:

+ Use the Octocat or GitHub logo to link to your GitHub profile (or GitHub)

+ Use the Octocat or GitHub logo to advertise that your product has built-in GitHub integration

+ Use the Octocat or GitHub logo in a blog post or news article about GitHub

 

PLEASE, DON"T DO THESE THINGS:

- Use the Octocat or GitHub logo for your application's icon

- Create a modified version of the Octocat or GitHub logo

- Integrate the Octocat or GitHub logo into your logo

- Use any GitHub artwork without permission

 

NAMING YOUR APPLICATION

Please avoid naming your projects anything that implies GitHub's endorsement. For example, don't call your awesome new iPhone app "GitHub Issues". Instead consider "Octoissues" or "HubIssues" or "Gissues". Maybe not that last one, but you get the idea.

 

和訳

Githubはサードパーティのディベロッパを愛しています。私たちはあなたのプロジェクトを素晴らしいものにする手助けをしたいと考えておりまして、画像によってGitHubで作られていることを示せるようにすることも手助けの一環です。すべての人がハッピーであるために（そして法的な問題を避けるために）、私たちはあなたがご自身のアプリケーション内で使用できる画像を管理するためのリポジトリを作成しました。

ここに含まれない用途で画像を使用する場合には、support@github.com にご連絡ください。

 

以下の用途での使用を推奨します。

+ あなたのGitHubプロフィールへのリンクのためのOctocatもしくはGitHubロゴの使用

+ あなたのプロダクトがGitHub上で作成されていることを示すためのOctocatもしくはGitHubロゴの使用

+ GitHubに関するブログ記事やニュース記事内でのOctocatもしくはGitHubロゴの使用

 

以下の用途では使用しないでください。

- あなたのアプリケーションアイコンとしてのOctocatもしくはGitHubロゴの使用

- 改変を加えてのOctocatもしくはGitHubロゴの使用

- あなたのロゴに含む形でのOctocatもしくはGitHubロゴの使用

- 他の許可のないGitHubの画像の使用

 

アプリケーションの命名について

GitHubがあなたのアプリケーションに保証を与えているような命名は避けてください。例えばあなたが作成したiPhoneアプリに "GitHub Issues" という名前をつけてはいけません。代わりに "OctoIssues", "HubIssues", "Gissues"といった名前を検討してください。そのどれもがピッタリとくる名前ではないかもしれませんが、きっとよい名前の候補が見つかるでしょう。

 

使用させていただきます

「GitHubに関するブログ記事やニュース記事内でのOctocatもしくはGitHubロゴの使用」はOKとのことですので、早速先日書いたCEO・Chris Wanstrathのインタビューの翻訳エントリにも画像を掲載させていただきました。
