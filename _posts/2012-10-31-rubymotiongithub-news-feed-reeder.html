Octofeedの概要

Github New Feedのリーダアプリです。（Github New Feedとはなんぞや？については後述）

Safariだと読みにくいREADMEが読みやすくなってたりします。

詳しくはリリースされてから書こうかと。

 

Octofeed作成の動機

21世紀にはいって干支も一周しまして、それはそれはたくさんのWeb系スタートアップが乱立しているわけですけども、Githubという企業は際立ってイデオロギー的と言いますか、哲学的と言いますか、従業員の働き方から思い描く未来像までしっかりはっきりと表明している企業なんですね。

成長のための重要事項は現場のモチベーションと考え、マネジメントなどというものは現場の邪魔をするだけのものなので不要だ、などと言ってしまうのです、CEOが。従来のCEO像からすると自己否定以外のなにものでもないとをサクッと言い放つあたりが素敵だな、と。

将来的にはプログラマだけでなく、医者や教師もコーディングをして本業に集中できる環境を自ら作るのだ、などとも言っていて、私などはほんまかいなと半信半疑だったりするのですが、昔々インターネッツにもっとみんな繋がるといい世の中になるのにね、と思っていたところ、いつの間にかスマートフォンなどというものが出現し、Facebookというものが出現し、ほとんどの人がインターネッツにつながるようになってしまってる現在があるわけですから、そんな未来もあるのかな、とも思うわけです。

と、のんびり構えてましたら、いつの間にやらこんなものを作り上げていて、おお本気じゃねーか、と肝がフルフルさせらりと、いろいろとエキサイティングな企業なのですよ、Githubというものは。

 

      GitHub Training

Education from the experts

We love helping people just like you use Git and GitHub to collaborate and be more productive. No matter what your level of experience, we’ve got something just for you.

 

 

そんなビジョンのもと、生産性を最重要視した現場が作り上げていくGithubには、それはそれはすごい勢いでソフトウェアのみならず、学術論文であったり、クリエイティブ・コモンな文献の翻訳であったりが日々追加されていっているわけですが、Githubはそれらを追いかけるための仕組みも、もちろん準備してそれがGithub News Feed と呼ばれるものです。

Github News Feedには以下の情報が含まれまして

  • Followしているユーザの
    • リポジトリ作成・更新・削除
    • Gitの作成・更新
    • リポジトリのFork
    • リポジトリのBookmark（スターをつける）
    • などなど
  • Watchしてるリポジトリの
    • Push / Pull Request / Merge
    • Pull Request のOpen / Close
    • Issueの投稿 / Issueへのコメント
    • Wikiの編集
    • などなど

 

 情報収集ツールとして利用する場合は、気になる人をひたすらFollowしていけば、その人のGithub上での情報が続々と流れこんでくる素敵なストーキングツールとして

リポジトリの改善に参加する場合は、その対象リポジトリをWatchしておけば、そのリポジトリの様々な活動が流れこんでAlertツールとして利用することができるわけです。

 

私はもっぱらストーキングツールとしてRSSリーダに登録して利用していたのですが、以下のことをするために1度Safariに飛ばねばならず、不便だな、と。

  • 流れてきたリポジトリのREADMEを読む
  • 流れてきたリポジトリにスターをつける
  • 流れてきたリポジトリのOwnerのプロフィールをみる
  • 流れてきたリポジトリのOwnerをFollowする

 

ええ、不満ドリブンです。

動機をまとめるとこの2点。

  • Github大好き♡
  • 既存RSSリーダだとGithub News Feedを追いかけるのにやや不便

あとついでにもうひとつ。

  • RubyMotionが気持ちよすぐる

 

Xcodeで作りかけのアプリがひたすら放置されてるんですけど、どうしましょ・・アイコン他人様に頼んでまで作ってもらったのに・・・
